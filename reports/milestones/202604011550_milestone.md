## Why This Matters

The capstone meeting feedback introduced confusion around where agents fit after supervised model training.
This milestone captures a repository-local explanation that separates:

- prediction with `CatBoost`
- action selection by an agent
- system-level evaluation through a reward function

The document is intentionally written for non-ML developers and should be the first reference when explaining the design to reviewers or teammates outside the ML subteam.

## Current Recommended Framing

- `CatBoost` is the predictor, not the full agent system
- The agent is the policy layer that decides actions from predicted risk plus current cluster state
- The reward function evaluates whether those actions improved the cluster while controlling side effects such as latency, migration cost, or SLA impact
